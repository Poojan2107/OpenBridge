# Security Policy

## Supported Versions

| Version | Supported                               |
| ------- | --------------------------------------- |
| 0.x     | :white_check_mark: (active development) |

## Reporting a Vulnerability

If you discover a security vulnerability in OpenBridge, please report it privately.

**Do not** open a public issue. Instead, email the maintainer directly or
create a [private advisory](https://github.com/Poojan2107/OpenBridge/security/advisories/new).

You should receive a response within 48 hours. If you don't, please follow up.

## What to include

- A description of the vulnerability
- Steps to reproduce (PoC preferred)
- Potential impact
- Suggested fix (optional but appreciated)

## Scope

- All source code (TypeScript, server, client)
- Dependencies (please check if the issue is upstream first)
- Build, CI/CD, and deployment configurations

## Preferences

We appreciate coordinated disclosure. Please:

1. Report privately
2. Allow time for a fix before public disclosure
3. Include "OpenBridge" in the subject line

Thank you for helping keep OpenBridge secure.
